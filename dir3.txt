方向三：用户认证与权限管理
 
📋 任务概述
 
在前序任务中，班级留言板允许任何用户匿名提交留言，无任何身份验证机制。这在实际生产环境中存在安全风险
（如恶意刷屏、冒充他人发言等）。
本次拓展任务要求你在现有留言板基础上，增加用户注册、登录和JWT身份认证功能，实现留言与用户的绑定，并
区分普通用户与管理员权限。
最终目标：掌握Web应用中的用户认证机制（JWT）、权限分级管理，以及前后端分离架构下的身份验证实现。
🎯 任务要求
 
在已完成班级留言板基础部署（前端 + 后端 + MySQL数据库）的Kubernetes集群中，完成以下功能拓展：
一、后端功能拓展
1. 用户数据模型：在MySQL数据库中新增 users 表，字段包括：id（自增主键）、username（唯一）、
password（需加密存储，使用bcrypt）、role（枚举类型：user / admin，默认 user）、created_at（时间
戳）。
2. 注册接口：新增 POST /api/auth/register 接口，接收 username 和 password，校验用户名是否已存在，密
码需加密后存入数据库。
3. 登录接口：新增 POST /api/auth/login 接口，验证用户名和密码，校验通过后签发 JWT Token（有效期24小
时），返回 token 和用户信息。
4. 认证中间件：编写JWT验证中间件，对需要登录的接口进行拦截校验（Authorization: Bearer <token>）。
5. 留言关联用户：改造 POST /api/messages 接口，在认证状态下自动将留言与当前登录用户绑定，在 
messages 表中新增 user_id 外键字段，关联 users 表。留言返回时携带留言人用户名。
6. 管理员权限：新增 DELETE /api/messages/:id 接口，要求只有 role=admin 的用户才能删除留言，普通用户
无权限。
7. 用户信息接口：新增 GET /api/auth/me 接口，返回当前登录用户的信息（用户名、角色）。
二、前端功能拓展
1. 注册/登录页面：新增注册和登录表单页面，支持用户名+密码的注册登录流程。
2. 登录状态管理：在前端实现Token存储（localStorage），并在每次API请求时自动携带 Authorization 头。
3. 留言展示优化：每条留言显示提交者的用户名，并在用户已登录时高亮显示自己的留言。
4. 删除权限控制：当登录用户角色为管理员时，每条留言旁显示删除按钮，点击后调用 DELETE 
/api/messages/:id 接口删除对应留言。
5. 登录状态UI反馈：页面右上角显示当前登录用户名，并提供退出登录按钮；未登录时显示“登录/注册”入口。
6. 注册/登录跳转：未登录用户尝试提交留言时，自动跳转到登录页面。
三、数据迁移与初始化
1. 编写数据库迁移脚本（migrate.sql），包含 users 表的创建和 messages 表的结构变更（新增 user_id 外
键）。
2. 使用ConfigMap挂载迁移脚本，通过Init Container或手动执行完成数据库迁移。

📌 说明
 
前置依赖：必须已完成班级留言板基础部署（前端 + 后端 + MySQL数据库），且MySQL有状态服务运行正常。

运行环境：基于 Docker Desktop 自带的 Kubernetes 集群（单节点，执行 kubectl get nodes 显示 docker-desktop 
节点状态为 Ready）。
JWT密钥：JWT签名密钥必须存放在Secret中，通过环境变量（JWT_SECRET）注入后端容器，禁止硬编码。
密码加密：用户密码必须使用bcrypt进行哈希加密存储，禁止明文存储。
前端路由：需配置前端路由（React Router / Vue Router），支持 /login、/register、/ 三个页面切换。
已有留言兼容：历史留言的 user_id 字段允许为NULL，前端展示时显示为“匿名用户”，不影响已有功能。
镜像构建：Docker Desktop为单节点集群，镜像构建后已在本地节点可用，无需额外分发。
完成本拓展方向 = 具备前后端分离架构中用户认证与权限管理的完整实现能力。 🚀
